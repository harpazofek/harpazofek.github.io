Polybot: AI-Powered Telegram Bot Deployed with AWS

The Polybot Project is a Telegram bot built with Python and integrated with a REST API utilizing the YOLOv5 AI model for image analysis.

  • Deployed on AWS using Terraform modules for scalability and fault tolerance across multiple Availability Zones.
  • Telegram bot integrates with AWS services like S3, SQS, and DynamoDB to handle image uploads, analysis, and results.
  • Dockerized infrastructure with Polybot and YOLOv5 instances.
  • CI/CD pipelines automate infrastructure provisioning and application deployment.

GCP MongoDB & GKE

This project deploys a highly available and fault-tolerant architecture on GCP using Terraform. It provisions a secure environment featuring a MongoDB replica set and a GKE cluster, each hosted on private subnets within a shared VPC.

  • Secure Architecture: Private subnets for MongoDB and GKE, with public access via Bastion Host.
  • Scalability: Horizontal Pod Autoscaler and spot node pools ensure cost-effective scaling.
  • Automation: Modularized Terraform code for efficient and reusable infrastructure provisioning.
  • Enhanced Access: TLS-based ingress and Identity-Aware Proxy (IAP) for secure application and Argo CD integration.

Octopus's Disko Image Management

Disko is an open source tool, designed to manage and facilitate operations in disconnected (air-gapped) environments. It has three main features: Statistics of Images per Registry, Copy Images Between Registries and Migrate Images in Kubernetes.

  • Transformed Disko's CLI to a Next.js web-based GUI, developed a seamless API with Python.
  • Implemented CI/CD pipelines using GitHub Actions for both frontend and backend, impacting deployment time.
  • Integrated Helm to automate the deployment of container images, streamlining the transfer between registries and clusters.
